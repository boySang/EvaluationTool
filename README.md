# EvaluationTool 等级保护测评辅助工具

EvaluationTool 是一款面向 Windows 现场测评工作的轻量桌面工具。当前体验版已经打通一个可测试的安全闭环：建立项目、保存 SSH Linux 服务器、人工确认主机指纹、进行无命令登录测试、执行内置只读命令，并保存原始输出、证据截图和执行记录。

> 当前仍是体验版，只用于验证首个真实采集闭环，不代表已经支持全部设备、数据库和中间件，也不能代替测评人员作出最终合规结论。

## 当前已实现

- Windows 10、Windows 11 x64 桌面程序，目标框架为 .NET Framework 4.8。
- 本地管理客户、项目、证据目录和设备连接资料。
- 添加 SSH 设备，支持密码或未加密的 PuTTY PPK v2/v3 私钥认证；凭据由当前 Windows 用户的 DPAPI 加密保存。
- 使用随体验包提供并经过构建流程校验的 PuTTY Plink 0.84 建立 SSH 连接。
- 首次连接先探测完整 SSH 主机指纹，由测评人员人工核对；确认后再次探测，再进行不执行远程命令的登录测试。
- 主机指纹变化时，在读取密码和自动登录前阻止连接。
- 对“自动识别”或“服务器”类别的 SSH Linux 设备执行内置只读闭环。
- 识别命令：`uname -a`、`cat /etc/os-release`。
- 每次形成唯一可靠的设备识别结果时，以追加式审计记录保存类别、厂商、产品系列、型号、版本、识别依据、可信度和人工确认来源；设备页显示最近一次记录，历史记录禁止覆盖或删除。
- 低可信或多个识别候选会先保存为追加式待确认批次；关闭软件或重启电脑后重新选择同一设备仍可恢复。确认旧候选时软件必须重新执行低风险识别并与当前结果逐项匹配，不能直接使用旧结果启动完整采集。
- 当前正式采集命令：`hostname`、`cat /etc/login.defs`。
- 自动采集不提供自由命令输入框，只使用固定哈希校验通过、标记为已验证且通过安全策略检查的内置命令。
- 每条已执行命令分别保存原始输出、分页 PNG 截图、`执行记录.json` 和 SHA-256，并写入本地 SQLite 证据索引。
- “证据中心”可查看当前项目的执行状态、命令、原始输出相对路径和截图数量，可安全打开项目证据目录，并可显式只读复算原始输出和截图的 SHA-256。
- 数据库候选的人工确认会作为追加式审计记录保存；软件重启后仍可在“证据中心”查看设备、产品、版本、发现依据、可信度、确认时间和确认来源。
- “命令库”可导入不超过 1 MB 的 UTF-8 JSON 草稿，保存原始 SHA-256、命令摘要和结构/风险发现；所有导入内容强制为“待校验、禁止执行”，不会进入正式命令包或连接会话。
- 支持浅色、深色主题，以及错误原因和处理建议展示。

## 下载 Windows 体验包

体验包由 GitHub Actions 的 Windows Runner 自动构建：

1. 打开仓库 [boySang/EvaluationTool](https://github.com/boySang/EvaluationTool)。
2. 进入 **Actions**，选择 **Windows CI**。
3. 打开 `main` 分支最近一次带绿色对勾的成功运行。
4. 在页面底部 **Artifacts** 区域下载 `EvaluationTool-Release-windows-x64`。
5. GitHub 下载的是压缩包。请先完整解压到普通文件夹，不要在压缩包预览窗口中直接运行。

构建产物默认保留 14 天。若页面要求登录，请使用有权访问该仓库的 GitHub 账号；不需要把账号密码提供给软件。

## 运行要求

- Windows 10 x64 或 Windows 11 x64。
- .NET Framework 4.8。
- 普通 Windows 用户权限即可运行；当前体验版不要求安装程序，也不应以管理员身份运行。
- 能够从测试电脑访问目标设备的 SSH 地址和端口。
- 仅使用已经获得客户明确授权的测试设备和最小权限测评账户。

完整解压后，运行根目录中的 `AssessmentTool.App.exe`。请保留程序目录的完整结构，尤其不要单独复制 EXE，也不要删除 `x64`、`依赖组件`、DLL 或配置文件。

若软件无法启动或 SSH 功能不可用：

1. 确认压缩包已经完整解压。
2. 确认系统为 x64，并已启用 .NET Framework 4.8。
3. 打开软件的“组件中心”，检查 `依赖组件\plink.exe` 是否通过可信组件检测。
4. 不要从非官方来源自行替换 Plink；当前版本不会自动下载或安装组件。

## 完整体验流程

### 1. 创建项目

1. 打开左侧“项目”。
2. 填写客户名称、项目名称和证据保存目录。
3. 单击“创建并保存项目”。
4. 在“已保存项目”中选择刚创建的项目。

证据保存目录应位于当前用户可写的位置。项目名称和设备名称会用于整理证据目录，软件会处理 Windows 非法字符、重名和过长路径。

### 2. 添加 SSH Linux 设备

1. 打开左侧“设备”，单击“添加设备”。
2. 填写设备名称、主机地址或 IP。
3. 设备类别选择“自动识别（推荐）”或“服务器”。
4. 填写 SSH 用户名和端口，并选择密码或 PuTTY PPK 私钥认证。
5. 核对信息后单击“保存设备”。

保存设备只写入本机项目资料和 DPAPI 加密凭据，不会连接客户设备，也不会执行远程命令。私钥认证仅接受不超过 256 KB、严格 UTF-8 编码、`Encryption: none` 的 PuTTY PPK v2/v3；界面和项目数据库不保存原始文件路径。带口令的 PPK 暂不支持，避免在批处理参数或日志中暴露口令。

### 3. 探测并确认 SSH 主机指纹

1. 在设备列表中选中设备。
2. 单击“探测主机指纹”。此步骤不会读取已保存密码，也不会执行远程命令。
3. 将界面显示的完整算法和指纹与客户管理员提供的可信值核对。
4. 只有完全一致时，单击“确认并测试登录”。

软件会再次探测指纹。如果与刚确认的值不一致，会立即阻止登录。不要在无法独立核实指纹时仅凭界面提示盲目确认。

### 4. 无命令登录测试

指纹复核通过后，软件按需读取 DPAPI 保护的密码或私钥并尝试建立 SSH 会话，但不向目标发送业务查询命令。私钥只会短暂写入当前用户严格权限控制的临时文件，并在使用后校验身份并清理。只有无命令登录测试成功后，当前设备才具备开始自动采集的条件。

登录失败时，请检查地址、端口、用户名、密码或私钥、网络和服务器 SSH 策略。错误凭据不会触发自动采集。

### 5. 开始只读采集

1. 打开左侧“采集任务”。
2. 确认当前项目、当前设备和连接状态正确。
3. 单击“开始只读采集”。
4. 软件先执行固定的 Linux 识别命令。
5. 如果识别结果需要人工确认，查看识别依据并选择“确认并重新识别”；软件重新执行低风险识别，只有当前结果仍与所选候选完全一致时才继续。
6. 确认后执行当前内置采集命令；运行中可单击“安全停止”。

采集只对 SSH Linux 服务器开放。其他类别即使能够保存设备或完成连接测试，也会在自动采集阶段被安全阻止。

### Linux 主机数据库实例发现（当前体验范围）

现有 SSH Linux 只读采集流程已接入“数据库实例发现”，首批范围严格限定为 PostgreSQL、MySQL 和 MariaDB。该能力只读取当前主机上的最少量元数据，用于形成候选实例，不代表已经支持所有数据库或所有版本。

计划读取的来源包括：

- 本机进程名称等最小只读信息，不采集可能包含密码的完整进程启动参数。
- systemd 当前服务的只读状态信息。
- Docker 或 Podman 的容器名称、镜像、端口等只读元数据；目标主机未安装对应容器工具时跳过该项，不影响其他发现步骤。

发现结果会显示产品候选、发现依据和可信度，并必须由测评人员人工确认。当前阶段只确认“主机上可能存在什么数据库实例”，不建立数据库直连、不读取数据库凭据、不执行任何 SQL，也不会因为发现到候选实例而自动运行数据库命令包。

### 6. 查看证据

1. 打开左侧“证据中心”。
2. 单击“刷新索引”，查看数据库人工确认审计、命令执行状态、原始输出相对路径、截图数量和 SQLite 索引状态。
3. 单击“复核文件 SHA-256”，让软件只读打开原始输出和截图并重新计算哈希。
4. 复核结果为“文件与索引 SHA-256 一致”时，表示本次读取到的文件内容与本地项目索引记录一致。
5. 单击“打开证据目录”，在 Windows 资源管理器中查看实际文件。

证据通常按“项目名称 / 设备名称 / 测评项 / 执行批次”整理。每条命令的目录包含：

- `原始输出.txt`：SSH 会话实际保存的标准输出和错误输出。
- `证据_*.png`：由同一份原始输出分页生成的可阅读截图。
- `执行记录.json`：命令、时间、状态、文件名和 SHA-256 等结构化记录。
- `待入库.txt`：仅在证据文件已保存、但 SQLite 索引写入失败时出现，表示该批次需要后续恢复索引。

“索引完整（未复核文件）”只表示 SQLite 中已记录原始输出和截图的哈希索引；只有用户显式单击“复核文件 SHA-256”后，软件才读取磁盘证据并复算哈希。该结果证明文件与本机索引一致，不等同于具有第三方数字签名的不可否认性证明。

数据库人工确认记录是对“谁在何时基于什么发现依据确认了哪个候选实例”的本地审计记录，不属于证据文件，因此不参与文件 SHA-256 复核，也不代表软件已经建立数据库直连或执行 SQL。

### 7. 导入命令草稿

1. 打开左侧“命令库”。
2. 单击“导入 JSON 草稿”，选择不超过 1 MB 的 UTF-8 JSON 文件。
3. 查看命令数量、原始文件 SHA-256、阻断项和校验发现。

导入是纯本地离线操作，不会连接设备、SSH、数据库或中间件，也不会执行文件中的命令。文件自称 `Verified` 或 `isReadOnly: true` 不会被信任；中高风险声明和明显修改语句会形成阻断项。当前版本没有发布按钮，导入草稿不能转换为正式可执行命令包。

## 隐私与安全

- 软件不会自动扫描客户网段，只连接用户手动添加的主机和端口。
- 软件不会自动上传客户资料、设备信息、密码、命令输出、截图或证据文件。
- 项目、设备和证据索引保存在 `%LOCALAPPDATA%\EvaluationTool\data\assessment.db`。
- SSH 密码或私钥以不透明凭据引用形式写入项目数据库，实际密文保存在 `%LOCALAPPDATA%\EvaluationTool\security` 下，并使用当前 Windows 用户的 DPAPI 保护；数据库和普通日志不保存私钥正文或原始路径。
- DPAPI 密文通常不能由其他 Windows 用户或复制到另一台电脑后直接解密。请勿把整个本地数据目录当作可迁移的明文备份。
- 证据文件保存在创建项目时由用户指定的本地目录，软件不会自动清理或上传。
- 自动采集没有自由 Shell 输入，只能运行受信任命令包中的固定只读命令。
- 即使登录账户具有管理员或 root 权限，软件的自动采集仍只允许白名单命令；仍建议使用只读或最小权限账户。
- 首次 SSH 指纹必须人工核对；指纹变化会阻止自动连接。

## 当前限制

以下能力尚未作为当前体验版的可用功能交付：

- Telnet、串口、带口令 SSH 私钥及 Pageant、跳板机、PowerShell、WinRM。
- Windows 服务器自动采集。
- 主流交换机、路由器和安全设备的正式命令包。
- Linux 主机上的 PostgreSQL、MySQL、MariaDB 实例只读发现已形成首个体验闭环；正式支持范围仍以版本说明和已验证适配清单为准。
- 数据库直连，以及 MySQL、PostgreSQL、MariaDB、Oracle、达梦、Kingbase、GaussDB、MongoDB 等数据库的正式查询闭环。
- Nginx、IIS、Tomcat、Apache 和国产中间件的正式采集闭环。
- 命令草稿已经支持安全导入、持久化和风险查看；编辑、人工审核工作流、官方资料校验、签名、发布、导出和回滚正式命令包仍未交付。
- 自动安装缺失组件、安装包部署、自动更新和在线升级。
- 直接打开单个证据文件、导出报告，以及带第三方数字签名的证据验签。
- 对 Windows 12 或 Windows on ARM 的正式兼容性验证。

仓库中可能包含尚未接入用户界面的研究代码、测试夹具或候选命令包；它们不等于当前体验版已经支持对应设备。

## 开发与验证

Windows CI 会执行核心逻辑测试、Windows 平台服务测试、Release x64 构建、SQLite x64 原生组件检查、可信 Plink 校验、打包后主窗口两次启动冒烟测试，以及发布目录 SHA-256 清单生成与复核。

体验包根目录中的 `SHA256SUMS.txt` 用于核对发布文件完整性，`版本与使用说明.txt` 记录对应 Git 提交和构建时间。真实设备兼容性仍需在明确授权和记录具体系统版本的条件下验证，不能从单一测试环境推断为支持所有 Linux 发行版或版本。

详细人工验证步骤见 [`docs/体验版测试清单.md`](docs/体验版测试清单.md)。
