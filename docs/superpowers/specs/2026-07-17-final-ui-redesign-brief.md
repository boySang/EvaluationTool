# 等级保护测评辅助工具：最终界面与使用流程设计说明

## 1. 文档用途

本说明用于指导界面设计师、产品经理或 AI 设计工具重新设计 Windows 桌面软件界面。

设计稿只描述最终产品体验，不代表所有功能已经开发完成。开发时仍按可验证的小版本逐步交付。

## 2. 产品定位

这是一款面向等级保护测评人员的 Windows 桌面辅助工具。它帮助用户管理项目和设备，以只读方式连接客户设备，自动识别设备信息，执行经过校验的查询命令，并保存原始输出、截图和审计记录。

软件不能替代测评人员作出最终合规结论，也不能执行修改、删除、重启、配置变更等操作。

## 3. 目标用户

- 熟悉等级保护测评业务，但不一定熟悉复杂软件配置。
- 经常在客户现场、弱网或无网环境中工作。
- 需要连接交换机、路由器、Windows/Linux 服务器、数据库和中间件。
- 希望减少重复输入命令、截图、命名文件和整理目录的工作。
- 需要明确知道软件当前正在做什么，以及是否对客户设备产生影响。

## 4. 核心设计原则

1. **业务语言优先**：使用“设备、采集、证据、连接测试”等名称，避免直接暴露底层技术名词。
2. **安全状态可见**：持续显示“只读模式”，每次执行前可以预览命令数量和用途。
3. **一步一件事**：连接和采集采用分步骤流程，避免把所有字段堆在一个页面。
4. **默认简单，高级可展开**：普通用户只看到必填内容，端口、编码、算法和代理等参数折叠显示。
5. **错误可处理**：错误提示必须回答“发生了什么、可能原因、怎么处理”。
6. **离线可用**：基础流程不依赖互联网，不使用需要实时加载的网页素材。
7. **结果可追溯**：成功、失败、跳过、人工确认和重新执行都有记录。

## 5. 最终完整使用流程

### 5.1 首次启动

1. 软件显示简短欢迎页。
2. 自动检查 Windows 版本、数据目录权限、PuTTY/Plink 和可选组件。
3. 必需组件正常时进入首页。
4. 可选组件缺失时不阻止启动，只提示受影响的功能和解决方法。
5. 用户选择默认项目保存位置和界面主题。

### 5.2 创建项目

1. 点击首页的“新建测评项目”。
2. 填写项目名称、客户名称、测评人员和证据保存目录。
3. 软件检查目录是否可写、路径是否过长。
4. 创建项目并进入项目工作台。

### 5.3 添加设备

1. 点击“添加设备”。
2. 填写设备名称、主机地址和连接方式。
3. 设备类别默认选择“自动识别”，也可预先选择网络设备、服务器、数据库或中间件。
4. 根据连接方式填写用户名、密码、私钥、串口或跳板机信息。
5. 凭据加密保存，界面明确提示密码不会明文保存。

### 5.4 安全连接测试

连接过程使用明确的三步向导：

1. **探测主机身份**：仅建立连接并读取 SSH 主机指纹，不输入密码、不执行命令。
2. **人工确认指纹**：首次连接展示指纹，请用户与客户提供的信息核对；指纹变化时阻止连接并高亮告警。
3. **无命令测试登录**：确认指纹后使用保存的凭据测试能否登录，不发送业务命令，也不发送回车。

连接成功后显示连接方式、响应时间、主机指纹、测试时间和“未执行任何命令”的说明。

### 5.5 自动识别与人工确认

1. 优先读取连接横幅、提示符和已有会话信息。
2. 执行最少量、经过验证的只读识别命令。
3. 显示对象类别、厂商、型号、系统或软件版本、识别依据和可信度。
4. 识别可信度高且结果唯一时，仍提供人工确认入口。
5. 结果冲突、多个实例或版本未知时，列出候选项，由用户选择。
6. 未确认的低可信结果不能启动完整采集。

### 5.6 创建采集任务

1. 用户选择一个或多个已确认设备。
2. 软件根据设备识别结果推荐适用的命令包。
3. 显示命令包版本、测评项数量、预计耗时、权限要求和无法覆盖的项目。
4. 用户可展开查看每条命令及其用途，但不能临时输入任意命令。
5. 点击“开始只读采集”后再次显示安全确认摘要。

### 5.7 执行只读采集

执行页面持续显示：

- 当前设备和连接状态。
- 当前测评项。
- 当前查询命令和用途。
- 已完成、失败、跳过和待确认数量。
- 当前耗时与总体进度。
- “安全停止”按钮。

遇到分页、超时、权限不足或结果不明确时，软件应暂停相应项目并给出处理建议，不把失败伪装成成功。

### 5.8 自动生成证据

每条命令完成后保存：

- 实际执行的命令。
- 原始文本输出。
- 可阅读截图；长输出自动生成连续分页截图。
- 开始时间、结束时间、状态和错误信息。
- 命令包版本、文件哈希和审计信息。

证据按“项目 / 设备 / 测评项”自动整理。

### 5.9 复核与导出

1. 用户进入证据中心查看全部证据。
2. 可按设备、测评项、成功状态和人工复核状态筛选。
3. 点击证据可并排查看命令、原始输出和截图。
4. 用户可以标记“已复核、需重采、人工补充”。
5. 导出前检查缺失证据、失败项目和哈希完整性。
6. 导出证据目录、执行清单和诊断摘要。

## 6. 软件整体导航

左侧使用可收起的 Fluent 风格导航栏：

1. **首页**：最近项目、待处理事项、组件状态和快捷入口。
2. **项目**：客户与项目列表、项目概况和项目设置。
3. **设备**：设备清单、连接资料、识别结果和连接测试。
4. **采集任务**：任务创建、执行进度、失败重试和历史记录。
5. **证据中心**：原始输出、截图、完整性和复核状态。
6. **命令库**：正式命令包、待校验命令、版本和导入导出。
7. **组件中心**：PuTTY、串口、数据库插件等依赖状态。
8. **设置**：主题、目录、日志、更新和安全选项。

导航栏底部固定显示：

- 当前版本。
- 离线或联网状态。
- 全局“只读保护已开启”状态。

## 7. 主要页面设计

### 7.1 首页

顶部欢迎区显示当前项目和一句明确引导，不使用大面积装饰图片。

核心内容：

- “新建项目”“打开最近项目”“导入项目”三个主要入口。
- 最近项目列表，显示客户、设备数量、最近采集时间和进度。
- 待处理事项：未确认指纹、识别待确认、采集失败、证据待复核。
- 环境状态：核心组件正常或有几项需要处理。

### 7.2 项目工作台

使用项目概览卡片和任务时间线：

- 项目名称、客户、负责人和证据目录。
- 设备总数、已连接、已识别、已采集数量。
- 最近执行记录。
- 缺失证据和失败项目。
- “添加设备”和“创建采集任务”作为主要按钮。

### 7.3 设备列表

采用表格与卡片相结合的方式：

- 设备名称、地址、类别、厂商型号、连接方式、连接状态、识别状态。
- 支持搜索和按状态筛选。
- 状态使用文字加颜色，不能只依赖颜色区分。
- 选中设备后右侧显示详情抽屉，不频繁跳转页面。

### 7.4 添加或编辑设备

使用分组表单：

- 基本资料：名称、地址、类别。
- 连接方式：SSH、Telnet、串口、Windows 远程、数据库直连等。
- 登录资料：用户名、密码或密钥。
- 高级设置：端口、编码、超时、跳板机、兼容算法。

底部固定操作栏：

- 取消。
- 保存设备。
- 保存并测试连接。

### 7.5 连接与识别页

页面中心使用三步步骤条：

`探测主机身份 → 确认指纹 → 测试登录`

完成登录测试后进入第二组步骤：

`读取基础信息 → 显示识别候选 → 人工确认`

右侧安全说明区持续显示：

- 当前操作是否会发送命令。
- 是否会读取密码。
- 是否会修改客户设备。
- 已确认的主机指纹。

### 7.6 创建采集任务页

采用四步向导：

1. 选择设备。
2. 确认识别结果和命令包。
3. 预览采集范围与只读命令。
4. 确认并开始。

最后一步必须明确显示：

- 总设备数。
- 总测评项数。
- 预计耗时。
- 只读策略检查结果。
- 将要生成的证据目录。

### 7.7 任务执行页

左侧是设备和测评项树，中间是当前终端输出，右侧是执行详情。

顶部显示总体进度和安全状态，底部显示运行日志和“安全停止”。

终端区域不是自由终端，不允许用户直接输入任意命令。

### 7.8 证据中心

顶部提供项目、设备、测评项、状态和日期筛选。

中间使用证据列表，右侧预览区包含三个标签：

- 截图。
- 原始输出。
- 执行与完整性信息。

支持放大截图、复制脱敏后的文本、打开所在目录和重新采集。

### 7.9 命令库

将“正式可执行”和“待校验”明确分区：

- 正式命令包显示签名或哈希、版本、适用范围和验证日期。
- 用户新增命令默认进入待校验区，并显示“禁止自动执行”。
- 命令详情显示用途、适用版本、权限、超时和来源。

### 7.10 组件中心

每个组件显示：

- 已安装、缺失、版本过低或检测失败。
- 影响哪些功能。
- 下载或离线安装包大小。
- 是否联网和是否需要管理员权限。
- “立即安装、选择离线包、查看步骤、暂不安装”。

## 8. 视觉风格建议

### 8.1 整体方向

- Windows 11 Fluent 风格，但在 Windows 10 上保持一致可用。
- 专业、可靠、克制，不使用炫目渐变和大面积插画。
- 以浅灰背景、白色内容卡片和蓝色主色为基础。
- 圆角适中，阴影非常轻，强调信息层级而不是装饰。

### 8.2 推荐颜色

- 主色：`#1677C8`，用于主要按钮和选中状态。
- 成功：`#15803D`。
- 警告：`#B45309`。
- 错误：`#C62828`。
- 信息：`#2563EB`。
- 页面背景：`#F5F7FA`。
- 卡片背景：`#FFFFFF`。
- 主文字：`#18212F`。
- 次要文字：`#667085`。
- 边框：`#D8DEE8`。

### 8.3 排版与尺寸

- 默认设计画布：1440 × 900。
- 同时验证 1366 × 768 和 1920 × 1080。
- 中文字体优先使用“微软雅黑”或 Windows 系统 UI 字体。
- 页面标题 24–28 px，卡片标题 16–18 px，正文 14 px。
- 基础间距采用 8 px 倍数体系。
- 主要按钮高度 36–40 px，输入框高度 36–40 px。

### 8.4 状态组件

统一设计以下状态：

- 未连接、连接中、连接成功、连接失败。
- 待识别、识别中、待确认、已确认、识别冲突。
- 待执行、执行中、成功、失败、跳过、已停止。
- 已复核、需重采、人工补充。
- 只读保护开启、指纹变化警告、组件缺失。

## 9. 关键弹窗

至少需要设计以下弹窗：

1. SSH 首次连接指纹确认。
2. SSH 指纹变化阻止连接。
3. 保存凭据说明。
4. 开始只读采集确认。
5. 安全停止任务确认。
6. 缺少组件与安装方式选择。
7. 错误详情与复制诊断信息。
8. 导出前缺失证据检查。

## 10. 空状态、加载状态和错误状态

每个页面都要设计：

- 没有数据时的空状态和下一步按钮。
- 加载时的骨架屏或进度状态。
- 操作成功后的轻量反馈。
- 网络中断、密码错误、权限不足、组件缺失和未知错误状态。
- 用户可以继续处理的明确操作，不只显示错误代码。

## 11. 设计稿交付清单

建议至少设计以下 12 个桌面页面：

1. 首次启动与环境检查。
2. 首页。
3. 新建项目。
4. 项目工作台。
5. 设备列表。
6. 添加或编辑设备。
7. SSH 指纹确认与无命令登录测试。
8. 自动识别与人工确认。
9. 创建采集任务。
10. 任务执行。
11. 证据中心与证据预览。
12. 组件中心。

同时提供：

- 浅色主题主要页面。
- 深色主题至少一个代表页面。
- 颜色、字体、间距和图标规范。
- 按钮、输入框、卡片、表格、步骤条、状态标签、弹窗和通知组件。
- 关键流程的可点击原型。

## 13. 设计验收标准

- 第一次使用的人能在没有说明书的情况下找到“新建项目、添加设备、测试连接”。
- 用户能清楚区分“保存设备、测试连接、识别设备、开始采集”。
- 用户始终知道当前操作是否会执行命令以及是否只读。
- 连接失败时，用户能直接看到下一步处理办法。
- 1366 × 768 下不遮挡主要操作按钮。
- 不依赖鼠标悬停才能看到关键信息。
- 状态不仅依赖颜色，也有文字和图标。
- 键盘焦点、高 DPI、浅色和深色主题可用。
- 页面风格统一，不出现不同组件库拼接的视觉差异。
