## 实现时必须修正

- 产品名称统一为“等级保护测评辅助工具”。
- 导航统一为：首页、项目、设备、采集任务、证据中心、命令库、组件中心、设置。
- 首页不显示尚未选中的具体设备。
- 不提供内网扫描、漏洞扫描、新设备自动注册或自由命令输入。
- “资产”统一改为“设备”，“审计”统一使用准确的“测评、采集、复核”等业务名称。
- 添加设备采用三步向导：基本信息、连接配置、保存与连接测试。
- SSH 流程严格为：自动探测指纹、人工确认、无命令测试登录。
- Telnet 必须提示明文传输风险，不能显示“全程加密”。
- 凭据可选择仅本次使用，或使用 Windows DPAPI 加密保存。
- 不宣称软件或输入内容自动“符合等保 2.0”；软件只提供测评辅助和证据采集能力。
- 所有页面以 1440×900 为主要设计尺寸，并验证 1366×768 下的主要操作可见性。

## 界面改造顺序

1. 建立颜色、字体、间距、按钮、卡片和状态标签等公共样式。
2. 改造应用外壳、左侧导航、顶部项目栏和只读状态。
3. 实现首页工作台。
4. 实现设备列表与设备详情。
5. 将添加设备改为三步向导。
6. 实现 SSH 指纹确认和无命令登录测试界面。
7. 补充加载、空状态、失败提示、键盘焦点和高 DPI 验证。
